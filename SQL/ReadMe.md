## 📌 Project Overview

Most fitness apps are generic. This system is **personalized** — it knows your city's weather, tracks what you ate and how long you slept, and combines all of it to give you a recommendation that feels like it came from a real trainer.

**Core idea:** Pull live weather from OpenWeatherMap API → store in Oracle DB → PL/SQL cursor joins it with workout, meal, sleep, and goal data → generates a personalized daily recommendation per user.

## 🗃️ Database Schema

All tables are created under **SYSTEM** schema on **XEPDB1** (Oracle XE Pluggable Database).

### Tables

| Table           | Description                                      |
|-----------------|--------------------------------------------------|
| `USERS`         | User profiles with biometrics and city location  |
| `WORKOUTS`      | Workout logs (type, duration, calories burned)   |
| `MEALS`         | Meal logs (calories, protein, carbs, fats)       |
| `SLEEP_LOG`     | Daily sleep hours per user                       |
| `GOALS`         | Fitness goals with start/end dates               |
| `WEATHER`       | Live weather data per city (via Python API)      |
| `RECOMMENDATIONS` | AI-like recommendations generated by cursor   |

### ER Relationships

```
USERS (1) ──< WORKOUTS      (user_id FK)
USERS (1) ──< MEALS         (user_id FK)
USERS (1) ──< SLEEP_LOG     (user_id FK)
USERS (1) ──< GOALS         (user_id FK)
USERS (1) ──< RECOMMENDATIONS (user_id FK)
USERS.location ──── WEATHER.location  (logical join)
```

### Normalization
The schema is normalized to **3NF**:
- No repeating groups
- All non-key attributes depend only on the primary key
- No transitive dependencies

## 🌦️ Weather API Integration

The Python script `Weather_api.py` fetches **live weather** for each user's city and inserts it into the `WEATHER` table.

**How it works:**
1. Reads all user locations from `USERS` table
2. Calls OpenWeatherMap API for each city
3. Extracts temperature, humidity, and condition
4. Inserts a row into `WEATHER` with `weather_seq.NEXTVAL`

**Run it:**
```bash
python Weather_api.py
```

**Sample output:**
```
City: Ahmedabad | Temp: 33.23°C | Humidity: 23% | Condition: Clear
City: Mumbai    | Temp: 31.50°C | Humidity: 58% | Condition: Clouds
City: Delhi     | Temp: 28.10°C | Humidity: 40% | Condition: Haze
City: Surat     | Temp: 34.00°C | Humidity: 20% | Condition: Clear
Weather inserted successfully for all users!
```

## 🤖 PL/SQL Cursors

### 1. Recommendation Cursor (Personal Trainer)
**File:** `cursors.sql`

Loops through every user and generates a personalized message based on:

| Factor               | Logic Example                                              |
|----------------------|------------------------------------------------------------|
| 😴 Sleep             | < 6 hrs → "Rest is critical, do light yoga today"         |
| 🌡️ Weather           | > 38°C → "Too hot, avoid outdoor exercise"                |
| 🏃 Yesterday's workout | No workout → "Don't skip days!"                         |
| 🍽️ Yesterday's meals  | > 2500 kcal → "Eat clean today"                          |
| 🎯 Goal              | Weight Loss → "Focus on cardio + calorie deficit"         |

Recommendations are saved to the `RECOMMENDATIONS` table.

### 2. Goal Progress Tracker
Calculates % of goal timeline completed and gives a status message (Just Started / Halfway / Almost There / Expired).

### 3. Workout Streak Counter
Checks the last 7 days for consecutive workouts and displays the current streak with motivational messages.

## 🚀 How to Run

### Prerequisites
- Oracle Database XE installed
- Oracle Instant Client 23c extracted
- Python 3.x with packages: `oracledb`, `requests`
- VS Code with Oracle SQL Developer Extension
- OpenWeatherMap free API key

### Setup Steps

**1. Configure credentials**
Create `config.py`:
```python
API_KEY     = "your_openweathermap_api_key"
DB_USER     = "SYSTEM"
DB_PASSWORD = "your_oracle_password"
DB_DSN      = "localhost:1521/XEPDB1"
```

**2. Create tables**
Connect to XEPDB1 as SYSTEM in VS Code and run:
```
SQL/create_tables.sql
```

**3. Insert sample data**
```
SQL/insert_sample_data.sql
```

**4. Fetch live weather**
```bash
python Weather_api.py
```

**5. Run cursors**
```
SQL/cursors.sql
```

**6. Run views and queries**
```
SQL/views.sql
SQL/queries.sql
```

- `config.py` is listed in `.gitignore` — never commit API keys or passwords
